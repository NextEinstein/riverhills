SourceID como plan alternativo

En los datos IMS, el campo <sourcedid> representa el código persistente ID de una persona tal como es usado en el sistema fuente. El campo <userid> es un campo separado que debería contener el código ID usado por el usuario cuando accede al sistema.

En muchos casos (no siempre) estos dos códigos podrían ser el mismo.

Algunos sistemas de información sobre estudiantes dan una salida errónea en el campo <userid>. Si éste fuera el caso, usted debería activar este ajuste, para poder usar el <sourcedid> como el ID de usuario de Moodle.

En caso contrario, deje esta opción sin activar.
